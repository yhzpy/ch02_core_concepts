> 适用版本：LVGL v8.4.x | 目标平台：i.MX6ULL (Cortex-A7, Linux)
> 本手册为快速参考文档，涵盖事件系统、样式系统、对象树与常用代码模板。

## 1. 事件系统速查

### 1.1 事件注册与删除

#### lv_obj_add_event_cb()

```c
lv_event_cb_t * lv_obj_add_event_cb(lv_obj_t * obj,
                                     lv_event_cb_t event_cb,
                                     lv_event_code_t filter,
                                     void * user_data);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `obj` | `lv_obj_t *` | 要监听事件的对象 |
| `event_cb` | `lv_event_cb_t` | 回调函数指针，签名为 `void cb(lv_event_t * e)` |
| `filter` | `lv_event_code_t` | 事件码过滤器，`LV_EVENT_ALL` 表示接收所有事件 |
| `user_data` | `void *` | 用户自定义数据指针，回调中通过 `lv_event_get_user_data()` 获取 |

**返回值**：事件描述符指针（可用于后续删除），失败返回 `NULL`。

#### lv_obj_remove_event_cb()

```c
bool lv_obj_remove_event_cb(lv_obj_t * obj, lv_event_cb_t event_cb);
```

- 移除对象上匹配的第一个事件回调。
- 如果同一回调注册了多次，只移除第一个。

#### lv_obj_remove_event_cb_with_user_data()

```c
bool lv_obj_remove_event_cb_with_user_data(lv_obj_t * obj,
                                            lv_event_cb_t event_cb,
                                            void * user_data);
```

- 精确匹配回调函数 + user_data 进行移除。
- 适用于同一回调绑定到多个对象的场景。

#### 代码模板

```c
/* 注册事件 */
static void btn_event_cb(lv_event_t * e)
{
    lv_event_code_t code = lv_event_get_code(e);
    lv_obj_t * btn = lv_event_get_target(e);

    if(code == LV_EVENT_CLICKED) {
        LV_LOG_USER("按钮被点击");
    }
}

void setup_button(void)
{
    lv_obj_t * btn = lv_btn_create(lv_scr_act());
    lv_obj_add_event_cb(btn, btn_event_cb, LV_EVENT_ALL, NULL);
}

/* 删除事件 */
void cleanup_button(lv_obj_t * btn)
{
    lv_obj_remove_event_cb(btn, btn_event_cb);
}
```

> **提示**：对象被 `lv_obj_del()` 删除时，其上所有事件回调会自动清除，无需手动移除。

### 1.2 事件码完整列表

#### 输入事件

| 事件码 | 说明 | 触发时机 |
|--------|------|----------|
| `LV_EVENT_PRESSED` | 对象被按下 | 手指/鼠标按下瞬间 |
| `LV_EVENT_PRESSING` | 对象持续被按压 | 按住期间每帧触发 |
| `LV_EVENT_PRESS_LOST` | 按压丢失 | 手指滑出对象区域后松开 |
| `LV_EVENT_RELEASED` | 对象被释放 | 手指/鼠标松开瞬间 |
| `LV_EVENT_SHORT_CLICKED` | 短按（未触发长按） | 快速按下并松开 |
| `LV_EVENT_LONG_PRESSED` | 长按触发 | 按住超过阈值时间（默认400ms） |
| `LV_EVENT_LONG_PRESSED_REPEAT` | 长按重复 | 长按后每隔一段时间重复触发 |
| `LV_EVENT_CLICKED` | 点击完成 | 按下后在同一对象上松开 |
| `LV_EVENT_DOUBLE_CLICKED` | 双击 | 短时间内连续两次点击 |
| `LV_EVENT_TRIPLE_CLICKED` | 三击 | 短时间内连续三次点击 |

#### 值变化事件

| 事件码 | 说明 | 触发时机 |
|--------|------|----------|
| `LV_EVENT_VALUE_CHANGED` | 值发生改变 | Slider/Checkbox/Dropdown等值变化时 |

#### 绘制事件

| 事件码 | 说明 | 触发时机 |
|--------|------|----------|
| `LV_EVENT_DRAW_MAIN_BEGIN` | 主绘制开始 | 对象主体绘制前 |
| `LV_EVENT_DRAW_MAIN` | 主绘制中 | 对象主体绘制时（可添加自定义绘制） |
| `LV_EVENT_DRAW_MAIN_END` | 主绘制结束 | 对象主体绘制后 |
| `LV_EVENT_DRAW_POST_BEGIN` | 后绘制开始 | 子对象绘制前 |
| `LV_EVENT_DRAW_POST` | 后绘制中 | 子对象绘制时 |
| `LV_EVENT_DRAW_POST_END` | 后绘制结束 | 子对象绘制后 |
| `LV_EVENT_DRAW_PART_BEGIN` | 部件绘制开始 | 每个绘制部件开始前 |
| `LV_EVENT_DRAW_PART_END` | 部件绘制结束 | 每个绘制部件结束后 |
| `LV_EVENT_COVER_CHECK` | 覆盖检查 | 判断对象是否完全覆盖某区域 |
| `LV_EVENT_REFR_EXT_DRAW_SIZE` | 刷新扩展绘制区域 | 阴影/轮廓超出对象边界时 |

#### 布局与尺寸事件

| 事件码 | 说明 | 触发时机 |
|--------|------|----------|
| `LV_EVENT_SIZE_CHANGED` | 尺寸改变 | 对象宽/高发生变化后 |
| `LV_EVENT_STYLE_CHANGED` | 样式改变 | 对象样式属性被修改后 |
| `LV_EVENT_LAYOUT_CHANGED` | 布局改变 | 对象或其父对象布局重新计算后 |

#### 生命周期事件

| 事件码 | 说明 | 触发时机 |
|--------|------|----------|
| `LV_EVENT_CREATE` | 对象创建 | `lv_xxx_create()` 完成后立即触发 |
| `LV_EVENT_DELETE` | 对象删除 | `lv_obj_del()` 执行时触发（对象仍有效） |
| `LV_EVENT_CHILD_CREATED` | 子对象创建 | 子对象创建后在父对象上触发 |
| `LV_EVENT_CHILD_DELETED` | 子对象删除 | 子对象删除后在父对象上触发 |
| `LV_EVENT_CHILD_CHANGED` | 子对象变化 | 子对象尺寸/位置变化后在父对象上触发 |
| `LV_EVENT_SCREEN_UNLOAD_START` | 屏幕卸载开始 | 屏幕切换动画开始时 |
| `LV_EVENT_SCREEN_LOAD_START` | 屏幕加载开始 | 新屏幕加载动画开始时 |
| `LV_EVENT_SCREEN_LOADED` | 屏幕加载完成 | 新屏幕完全显示后 |
| `LV_EVENT_SCREEN_UNLOADED` | 屏幕卸载完成 | 旧屏幕完全移除后 |

#### 焦点事件

| 事件码 | 说明 | 触发时机 |
|--------|------|----------|
| `LV_EVENT_FOCUSED` | 获得焦点 | 对象获得输入焦点 |
| `LV_EVENT_DEFOCUSED` | 失去焦点 | 对象失去输入焦点 |
| `LV_EVENT_LEAVE` | 离开 | 对象失去焦点（含组内切换） |
| `LV_EVENT_ENTER` | 进入 | 对象获得焦点（含组内切换） |

#### 手势与滚动事件

| 事件码 | 说明 | 触发时机 |
|--------|------|----------|
| `LV_EVENT_GESTURE` | 手势识别 | 检测到上/下/左/右滑动手势 |
| `LV_EVENT_SCROLL_BEGIN` | 滚动开始 | 滚动动画开始 |
| `LV_EVENT_SCROLL_END` | 滚动结束 | 滚动动画结束 |
| `LV_EVENT_SCROLL` | 滚动中 | 滚动位置变化时 |

#### 按键事件

| 事件码 | 说明 | 触发时机 |
|--------|------|----------|
| `LV_EVENT_KEY` | 按键按下 | 键盘/编码器按键事件 |
| `LV_EVENT_CANCEL` | 取消 | ESC键或编码器长按 |
| `LV_EVENT_READY` | 确认/完成 | Enter键或编码器短按 |

#### 其他事件

| 事件码 | 说明 | 触发时机 |
|--------|------|----------|
| `LV_EVENT_REFRESH` | 刷新请求 | 请求对象重绘 |
| `LV_EVENT_GET_SELF_SIZE` | 获取自身尺寸 | 布局系统查询对象固有尺寸 |
| `LV_EVENT_HIT_TEST` | 碰撞检测 | 判断点击是否命中对象 |
| `LV_EVENT_ALL` | 所有事件 | 用作 filter 参数，接收全部事件 |

### 1.3 lv_event_t 数据获取 API 表

| API 函数 | 返回值 | 说明 |
|----------|--------|------|
| `lv_event_get_code(e)` | `lv_event_code_t` | 获取当前事件码 |
| `lv_event_get_target(e)` | `lv_obj_t *` | 获取事件最初发生的对象（冒泡时不变） |
| `lv_event_get_current_target(e)` | `lv_obj_t *` | 获取当前正在处理事件的对象 |
| `lv_event_get_user_data(e)` | `void *` | 获取注册时传入的 user_data |
| `lv_event_get_param(e)` | `void *` | 获取事件参数（如 VALUE_CHANGED 时为新值指针） |
| `lv_event_get_draw_part_dsc(e)` | `lv_obj_draw_part_dsc_t *` | 获取绘制部件描述符（DRAW_PART_BEGIN/END） |
| `lv_event_get_key(e)` | `uint32_t` | 获取按键值（KEY 事件） |
| `lv_event_get_scroll_dir(e)` | `lv_dir_t` | 获取滚动方向（SCROLL 事件） |
| `lv_event_get_gesture_dir(e)` | `lv_dir_t` | 获取手势方向（GESTURE 事件） |
| `lv_event_get_clip_area(e)` | `const lv_area_t *` | 获取裁剪区域（绘制事件） |
| `lv_event_get_old_size(e)` | `lv_area_t *` | 获取变化前的尺寸（SIZE_CHANGED） |
| `lv_event_get_indev(e)` | `lv_indev_t *` | 获取触发事件的输入设备 |

#### target vs current_target 区别

```
事件发生对象(target) → 父对象 → 祖父对象(current_target)
         ↑                              ↑
  lv_event_get_target()    lv_event_get_current_target()
  （始终指向最初对象）       （随冒泡层级变化）
```

#### 使用示例

```c
static void slider_event_cb(lv_event_t * e)
{
    lv_event_code_t code = lv_event_get_code(e);
    lv_obj_t * slider = lv_event_get_target(e);

    if(code == LV_EVENT_VALUE_CHANGED) {
        int32_t val = lv_slider_get_value(slider);
        LV_LOG_USER("滑块值: %d", val);
    }
}

static void key_event_cb(lv_event_t * e)
{
    if(lv_event_get_code(e) == LV_EVENT_KEY) {
        uint32_t key = lv_event_get_key(e);
        if(key == LV_KEY_ENTER) {
            LV_LOG_USER("Enter 被按下");
        }
    }
}

static void gesture_event_cb(lv_event_t * e)
{
    if(lv_event_get_code(e) == LV_EVENT_GESTURE) {
        lv_dir_t dir = lv_event_get_gesture_dir(e);
        switch(dir) {
            case LV_DIR_LEFT:  LV_LOG_USER("左滑"); break;
            case LV_DIR_RIGHT: LV_LOG_USER("右滑"); break;
            case LV_DIR_TOP:   LV_LOG_USER("上滑"); break;
            case LV_DIR_BOTTOM:LV_LOG_USER("下滑"); break;
        }
    }
}
```

### 1.4 事件冒泡

#### 启用冒泡

```c
lv_obj_add_flag(obj, LV_OBJ_FLAG_EVENT_BUBBLE);
```

#### 冒泡流程

```
用户点击子对象
      │
      ▼
┌─────────────┐
│  子对象回调  │  ← current_target = 子对象, target = 子对象
└──────┬──────┘
       │ (如果设置了 EVENT_BUBBLE)
       ▼
┌─────────────┐
│  父对象回调  │  ← current_target = 父对象, target = 子对象
└──────┬──────┘
       │ (如果父对象也设置了 EVENT_BUBBLE)
       ▼
┌─────────────┐
│ 祖父对象回调 │  ← current_target = 祖父对象, target = 子对象
└─────────────┘
```

#### 停止冒泡

```c
static void child_event_cb(lv_event_t * e)
{
    /* 阻止事件继续向父对象冒泡 */
    lv_event_stop_bubbling(e);
}
```

#### 完整冒泡示例

```c
void bubble_demo(void)
{
    lv_obj_t * parent = lv_obj_create(lv_scr_act());
    lv_obj_set_size(parent, 200, 200);
    lv_obj_add_flag(parent, LV_OBJ_FLAG_EVENT_BUBBLE);
    lv_obj_add_event_cb(parent, parent_cb, LV_EVENT_ALL, NULL);

    lv_obj_t * child = lv_btn_create(parent);
    lv_obj_set_size(child, 80, 40);
    lv_obj_add_flag(child, LV_OBJ_FLAG_EVENT_BUBBLE);
    lv_obj_add_event_cb(child, child_cb, LV_EVENT_ALL, NULL);
}

static void child_cb(lv_event_t * e)
{
    LV_LOG_USER("子对象收到: %d", lv_event_get_code(e));
    /* 不调用 stop_bubbling，事件继续冒泡到 parent */
}

static void parent_cb(lv_event_t * e)
{
    lv_obj_t * target = lv_event_get_target(e);
    lv_obj_t * cur = lv_event_get_current_target(e);
    LV_LOG_USER("父对象处理来自 %p 的事件", target);
}
```

> **注意**：冒泡仅对设置了 `LV_OBJ_FLAG_EVENT_BUBBLE` 的对象生效。默认情况下事件不冒泡。

### 1.5 手动发送事件

#### lv_event_send()

```c
lv_res_t lv_event_send(lv_obj_t * obj, lv_event_code_t event_code, void * param);
```

| 参数 | 说明 |
|------|------|
| `obj` | 目标对象 |
| `event_code` | 要发送的事件码 |
| `param` | 事件参数（可为 NULL） |

**返回值**：`LV_RES_OK` 成功，`LV_RES_INV` 对象已被删除。

#### 使用场景

```c
/* 场景1：程序模拟点击 */
lv_event_send(btn, LV_EVENT_CLICKED, NULL);

/* 场景2：通知值变化 */
int32_t new_val = 50;
lv_event_send(slider, LV_EVENT_VALUE_CHANGED, &new_val);

/* 场景3：自定义事件通信（使用 LV_EVENT_USER_1~4） */
typedef struct {
    int cmd;
    int data;
} my_msg_t;

my_msg_t msg = { .cmd = 1, .data = 100 };
lv_event_send(obj, LV_EVENT_USER_1, &msg);
```

> **提示**：`LV_EVENT_USER_1` ~ `LV_EVENT_USER_4` 是预留给用户的自定义事件码，可用于对象间通信。

### 1.6 事件处理最佳实践

#### 一个回调处理多个事件码

```c
static void multi_event_cb(lv_event_t * e)
{
    lv_event_code_t code = lv_event_get_code(e);
    lv_obj_t * obj = lv_event_get_target(e);

    switch(code) {
        case LV_EVENT_PRESSED:
            lv_obj_set_style_bg_color(obj, lv_color_hex(0x2196F3), 0);
            break;
        case LV_EVENT_RELEASED:
            lv_obj_set_style_bg_color(obj, lv_color_hex(0x1976D2), 0);
            break;
        case LV_EVENT_CLICKED:
            LV_LOG_USER("点击完成");
            break;
        case LV_EVENT_LONG_PRESSED:
            LV_LOG_USER("长按触发");
            break;
        default:
            break;
    }
}

/* 注册时接收所有事件 */
lv_obj_add_event_cb(btn, multi_event_cb, LV_EVENT_ALL, NULL);
```

#### user_data 传递结构体指针

```c
typedef struct {
    lv_obj_t * label;
    int counter;
    char name[32];
} btn_ctx_t;

static void btn_with_ctx_cb(lv_event_t * e)
{
    btn_ctx_t * ctx = (btn_ctx_t *)lv_event_get_user_data(e);
    ctx->counter++;
    lv_label_set_text_fmt(ctx->label, "%s: %d", ctx->name, ctx->counter);
}

void create_btn_with_context(void)
{
    static btn_ctx_t ctx = { .counter = 0 };
    strcpy(ctx.name, "计数器");

    lv_obj_t * btn = lv_btn_create(lv_scr_act());
    ctx.label = lv_label_create(btn);
    lv_label_set_text(ctx.label, "计数器: 0");

    lv_obj_add_event_cb(btn, btn_with_ctx_cb, LV_EVENT_CLICKED, &ctx);
}
```

> **警告**：user_data 指向的内存必须在对象生命周期内有效。避免使用局部变量地址。

#### 在回调中安全删除对象

```c
static void delete_self_cb(lv_event_t * e)
{
    lv_obj_t * obj = lv_event_get_target(e);

    /* 错误做法：直接删除正在处理事件的对象 */
    // lv_obj_del(obj);  /* 危险！可能导致崩溃 */

    /* 正确做法：异步删除 */
    lv_obj_del_async(obj);
}
```

#### 避免在回调中做耗时操作

```c
/* 错误：在事件回调中执行耗时 I/O */
static void bad_cb(lv_event_t * e)
{
    /* 不要这样做！会阻塞 UI 刷新 */
    FILE * f = fopen("/dev/mmcblk0", "r");
    fread(buf, 1, 4096, f);
    fclose(f);
}

/* 正确：设置标志，在定时器或主循环中处理 */
static volatile bool need_read = false;

static void good_cb(lv_event_t * e)
{
    need_read = true;  /* 仅设置标志 */
}

static void work_timer_cb(lv_timer_t * t)
{
    if(need_read) {
        need_read = false;
        /* 在此执行耗时操作 */
        do_file_read();
    }
}
```

## 2. 样式系统速查

### 2.1 样式 API 速查表

| API | 说明 |
|-----|------|
| `lv_style_init(&style)` | 初始化样式结构体（必须在使用前调用） |
| `lv_style_reset(&style)` | 重置样式，释放动态分配的内存 |
| `lv_obj_add_style(obj, &style, selector)` | 为对象添加全局样式 |
| `lv_obj_remove_style(obj, &style, selector)` | 移除对象上的指定样式 |
| `lv_obj_remove_style_all(obj)` | 移除对象上所有样式（恢复默认） |
| `lv_obj_refresh_style(obj, part, prop)` | 刷新对象样式（修改后调用） |
| `lv_obj_set_style_xxx(obj, value, selector)` | 设置局部样式属性 |
| `lv_obj_get_style_xxx(obj, part)` | 获取对象当前生效的样式属性值 |
| `lv_style_set_xxx(&style, value)` | 在全局样式中设置属性 |

#### 全局样式 vs 局部样式

```c
/* 全局样式：可复用于多个对象 */
static lv_style_t style_btn;
lv_style_init(&style_btn);
lv_style_set_bg_color(&style_btn, lv_color_hex(0x1976D2));
lv_style_set_radius(&style_btn, 8);

lv_obj_add_style(btn1, &style_btn, 0);
lv_obj_add_style(btn2, &style_btn, 0);  /* 复用同一样式 */

/* 局部样式：仅作用于单个对象 */
lv_obj_set_style_bg_color(btn1, lv_color_hex(0xF44336), 0);
```

### 2.2 Selector 详解

#### 基本规则

```
selector = part | state
```

- `part`：对象的哪个部分
- `state`：在什么状态下生效
- 默认 `selector = 0` 等价于 `LV_PART_MAIN | LV_STATE_DEFAULT`

#### Part 完整列表

| Part 常量 | 值 | 说明 | 适用对象 |
|-----------|-----|------|----------|
| `LV_PART_MAIN` | 0x000000 | 主体部分 | 所有对象 |
| `LV_PART_SCROLLBAR` | 0x010000 | 滚动条 | 可滚动容器 |
| `LV_PART_INDICATOR` | 0x020000 | 指示器 | Slider, Bar, Arc |
| `LV_PART_KNOB` | 0x030000 | 旋钮/把手 | Slider |
| `LV_PART_SELECTED` | 0x040000 | 选中项 | List, Dropdown, Roller |
| `LV_PART_ITEMS` | 0x050000 | 列表项 | Roller, Dropdown |
| `LV_PART_TICKS` | 0x060000 | 刻度 | Chart, Meter |
| `LV_PART_CURSOR` | 0x070000 | 光标 | Textarea, Chart |
| `LV_PART_TEXTAREA_PLACEHOLDER` | 0x080000 | 占位文本 | Textarea |
| `LV_PART_ANY` | 0x0F0000 | 任意部分 | 用于 remove_style |

#### State 完整列表

| State 常量 | 值 | 说明 |
|------------|-----|------|
| `LV_STATE_DEFAULT` | 0x0000 | 默认状态 |
| `LV_STATE_CHECKED` | 0x0001 | 选中/勾选状态 |
| `LV_STATE_FOCUSED` | 0x0002 | 获得焦点（触摸/点击） |
| `LV_STATE_FOCUS_KEY` | 0x0004 | 通过按键获得焦点 |
| `LV_STATE_EDITED` | 0x0008 | 编辑模式（编码器） |
| `LV_STATE_HOVERED` | 0x0010 | 鼠标悬停 |
| `LV_STATE_PRESSED` | 0x0020 | 被按压 |
| `LV_STATE_SCROLLED` | 0x0040 | 正在滚动 |
| `LV_STATE_DISABLED` | 0x0080 | 禁用状态 |
| `LV_STATE_USER_1` | 0x1000 | 用户自定义状态1 |

### 2.4 过渡动画（Transition）

#### lv_style_transition_dsc_t 结构

```c
typedef struct {
    const lv_style_prop_t * props;  /* 要过渡的属性数组，以 0 结尾 */
    lv_anim_path_cb_t path_xcb;    /* 动画路径函数 */
    uint32_t time;                  /* 过渡时间 (ms) */
    uint32_t delay;                 /* 延迟时间 (ms) */
    void * user_data;               /* 用户数据 */
} lv_style_transition_dsc_t;
```

#### 初始化函数

```c
void lv_style_transition_dsc_init(lv_style_transition_dsc_t * tr,
                                   const lv_style_prop_t props[],
                                   lv_anim_path_cb_t path_cb,
                                   uint32_t time,
                                   uint32_t delay,
                                   void * user_data);
```

#### 可用路径函数

| 路径函数 | 效果 |
|----------|------|
| `lv_anim_path_linear` | 线性（匀速） |
| `lv_anim_path_ease_in` | 缓入（先慢后快） |
| `lv_anim_path_ease_out` | 缓出（先快后慢） |
| `lv_anim_path_ease_in_out` | 缓入缓出 |
| `lv_anim_path_overshoot` | 过冲（超出后回弹） |
| `lv_anim_path_bounce` | 弹跳效果 |
| `lv_anim_path_step` | 阶跃（无过渡，直接跳变） |

#### 代码模板：按钮颜色过渡

```c
/* 定义要过渡的属性 */
static const lv_style_prop_t trans_props[] = {
    LV_STYLE_BG_COLOR, LV_STYLE_BG_OPA, 0  /* 0 表示结束 */
};

/* 过渡描述符 */
static lv_style_transition_dsc_t trans_def;  /* 默认状态过渡 */
static lv_style_transition_dsc_t trans_pr;   /* 按下状态过渡 */

void init_btn_transition(void)
{
    /* 默认状态：松开后 300ms 缓出回到原色 */
    lv_style_transition_dsc_init(&trans_def, trans_props,
                                  lv_anim_path_ease_out, 300, 0, NULL);

    /* 按下状态：按下时 150ms 线性变色 */
    lv_style_transition_dsc_init(&trans_pr, trans_props,
                                  lv_anim_path_linear, 150, 0, NULL);

    /* 默认状态样式 */
    static lv_style_t style_def;
    lv_style_init(&style_def);
    lv_style_set_bg_color(&style_def, lv_color_hex(0x1976D2));
    lv_style_set_transition(&style_def, &trans_def);

    /* 按下状态样式 */
    static lv_style_t style_pr;
    lv_style_init(&style_pr);
    lv_style_set_bg_color(&style_pr, lv_color_hex(0x0D47A1));
    lv_style_set_transition(&style_pr, &trans_pr);

    /* 应用到按钮 */
    lv_obj_t * btn = lv_btn_create(lv_scr_act());
    lv_obj_add_style(btn, &style_def, LV_STATE_DEFAULT);
    lv_obj_add_style(btn, &style_pr, LV_STATE_PRESSED);
}
```

> **关键**：过渡描述符和属性数组必须是 `static` 或全局变量，不能使用局部变量。

### 2.6 样式优先级规则

#### 优先级从高到低

```
1. 局部样式（lv_obj_set_style_xxx）     ← 最高优先级
2. 后添加的全局样式                      ← 同级别中后添加的优先
3. 先添加的全局样式
4. 主题样式（theme apply_cb）
5. 内置默认值                            ← 最低优先级
```

#### 同一样式中 Selector 优先级

```
更具体的 selector > 更通用的 selector

例如：
LV_STATE_PRESSED 的样式 > LV_STATE_DEFAULT 的样式
LV_PART_KNOB | LV_STATE_PRESSED > LV_PART_KNOB | LV_STATE_DEFAULT
```

#### 代码示例

```c
void priority_demo(void)
{
    lv_obj_t * btn = lv_btn_create(lv_scr_act());

    /* 优先级低：全局样式设置蓝色 */
    static lv_style_t style1;
    lv_style_init(&style1);
    lv_style_set_bg_color(&style1, lv_color_hex(0x2196F3));  /* 蓝色 */
    lv_obj_add_style(btn, &style1, 0);

    /* 优先级中：后添加的全局样式覆盖为绿色 */
    static lv_style_t style2;
    lv_style_init(&style2);
    lv_style_set_bg_color(&style2, lv_color_hex(0x4CAF50));  /* 绿色 */
    lv_obj_add_style(btn, &style2, 0);

    /* 优先级最高：局部样式覆盖为红色 */
    lv_obj_set_style_bg_color(btn, lv_color_hex(0xF44336), 0);  /* 红色 */

    /* 最终显示：红色（局部样式优先级最高） */
}
```

> **调试技巧**：使用 `lv_obj_get_style_bg_color(obj, LV_PART_MAIN)` 可获取当前生效的最终颜色值。

### 3.5 定时器 API 速查

| API | 说明 |
|-----|------|
| `lv_timer_create(cb, period_ms, user_data)` | 创建定时器 |
| `lv_timer_del(timer)` | 删除定时器 |
| `lv_timer_pause(timer)` | 暂停定时器 |
| `lv_timer_resume(timer)` | 恢复定时器 |
| `lv_timer_set_period(timer, ms)` | 修改周期 |
| `lv_timer_set_repeat_count(timer, cnt)` | 设置重复次数（-1=无限） |
| `lv_timer_ready(timer)` | 标记定时器立即就绪（下次循环执行） |
| `lv_timer_set_cb(timer, cb)` | 修改回调函数 |
| `lv_timer_get_next(timer)` | 获取下一个将触发的定时器 |

#### 定时器回调签名

```c
typedef void (*lv_timer_cb_t)(lv_timer_t * timer);
```

#### 基本用法

```c
static void my_timer_cb(lv_timer_t * t)
{
    /* 通过 user_data 获取上下文 */
    lv_obj_t * label = (lv_obj_t *)t->user_data;

    static int count = 0;
    lv_label_set_text_fmt(label, "计数: %d", count++);
}

void start_timer(void)
{
    lv_obj_t * label = lv_label_create(lv_scr_act());
    lv_obj_center(label);

    /* 每 1000ms 执行一次，无限重复 */
    lv_timer_t * timer = lv_timer_create(my_timer_cb, 1000, label);

    /* 5秒后停止 */
    // lv_timer_set_repeat_count(timer, 5);
}
```

> **注意**：定时器回调在 `lv_timer_handler()` 中执行，与 UI 刷新在同一线程。不要在回调中阻塞。

### 4.4 事件冒泡处理模板

```c
/* 场景：列表中的多个按钮，通过父容器统一处理 */

static void list_btn_cb(lv_event_t * e)
{
    lv_obj_t * btn = lv_event_get_target(e);
    int idx = (int)(intptr_t)lv_event_get_user_data(e);
    LV_LOG_USER("列表项 %d 被点击", idx);
}

static void container_cb(lv_event_t * e)
{
    lv_event_code_t code = lv_event_get_code(e);
    lv_obj_t * target = lv_event_get_target(e);
    lv_obj_t * current = lv_event_get_current_target(e);

    if(target == current) return;  /* 忽略容器自身的事件 */

    if(code == LV_EVENT_CLICKED) {
        int idx = lv_obj_get_index(target);
        LV_LOG_USER("容器捕获：子对象 %d 被点击", idx);
    }
}

void create_bubble_list(void)
{
    /* 容器 */
    lv_obj_t * container = lv_obj_create(lv_scr_act());
    lv_obj_set_size(container, 200, 300);
    lv_obj_align(container, LV_ALIGN_CENTER, 0, 0);
    lv_obj_set_flex_flow(container, LV_FLEX_FLOW_COLUMN);
    lv_obj_set_style_pad_all(container, 10, 0);
    lv_obj_set_style_pad_row(container, 8, 0);

    /* 容器监听冒泡事件 */
    lv_obj_add_flag(container, LV_OBJ_FLAG_EVENT_BUBBLE);
    lv_obj_add_event_cb(container, container_cb, LV_EVENT_ALL, NULL);

    /* 创建多个按钮 */
    for(int i = 0; i < 5; i++) {
        lv_obj_t * btn = lv_btn_create(container);
        lv_obj_set_width(btn, LV_PCT(100));
        lv_obj_set_height(btn, 40);
        lv_obj_add_flag(btn, LV_OBJ_FLAG_EVENT_BUBBLE);  /* 启用冒泡 */

        lv_obj_t * label = lv_label_create(btn);
        lv_label_set_text_fmt(label, "项目 %d", i + 1);
        lv_obj_center(label);

        /* 每个按钮也有自己的回调 */
        lv_obj_add_event_cb(btn, list_btn_cb, LV_EVENT_CLICKED,
                           (void *)(intptr_t)i);
    }
}
```

### 4.5 定时器 + UI 更新模板

```c
typedef struct {
    lv_obj_t * label_time;
    lv_obj_t * bar_progress;
    lv_timer_t * timer;
    int progress;
    int seconds;
} ui_ctx_t;

static ui_ctx_t g_ui;

static void ui_update_timer_cb(lv_timer_t * t)
{
    ui_ctx_t * ctx = (ui_ctx_t *)t->user_data;

    /* 更新进度条 */
    ctx->progress += 2;
    if(ctx->progress > 100) ctx->progress = 0;
    lv_bar_set_value(ctx->bar_progress, ctx->progress, LV_ANIM_ON);

    /* 更新时间标签 */
    ctx->seconds++;
    int min = ctx->seconds / 60;
    int sec = ctx->seconds % 60;
    lv_label_set_text_fmt(ctx->label_time, "%02d:%02d", min, sec);
}

void create_timer_ui(void)
{
    /* 进度条 */
    g_ui.bar_progress = lv_bar_create(lv_scr_act());
    lv_obj_set_width(g_ui.bar_progress, 200);
    lv_obj_align(g_ui.bar_progress, LV_ALIGN_CENTER, 0, -30);

    /* 时间标签 */
    g_ui.label_time = lv_label_create(lv_scr_act());
    lv_label_set_text(g_ui.label_time, "00:00");
    lv_obj_align(g_ui.label_time, LV_ALIGN_CENTER, 0, 20);

    /* 初始化上下文 */
    g_ui.progress = 0;
    g_ui.seconds = 0;

    /* 创建定时器：每 100ms 更新进度，每 1000ms 更新时间 */
    g_ui.timer = lv_timer_create(ui_update_timer_cb, 100, &g_ui);
}

void stop_timer_ui(void)
{
    if(g_ui.timer) {
        lv_timer_del(g_ui.timer);
        g_ui.timer = NULL;
    }
}
```

### 4.6 安全删除对象模板

```c
/* 场景：点击按钮后删除整个面板 */

typedef struct {
    lv_obj_t * panel;
    lv_obj_t * confirm_btn;
} del_ctx_t;

static void confirm_del_cb(lv_event_t * e)
{
    del_ctx_t * ctx = (del_ctx_t *)lv_event_get_user_data(e);

    /* 方法1：异步删除（推荐） */
    lv_obj_del_async(ctx->panel);

    /* 注意：此回调执行完毕后，ctx 指向的 btn 已无效 */
    /* 不要再访问 ctx->confirm_btn */
}

static void panel_close_cb(lv_event_t * e)
{
    lv_obj_t * panel = lv_event_get_target(e);

    /* 方法2：使用 DELETE 事件做清理 */
    LV_LOG_USER("面板即将被删除，执行资源清理");

    /* 释放关联的动态内存 */
    void * data = lv_obj_get_user_data(panel);
    if(data) {
        free(data);
    }
}

void create_deletable_panel(void)
{
    static del_ctx_t ctx;

    /* 面板 */
    ctx.panel = lv_obj_create(lv_scr_act());
    lv_obj_set_size(ctx.panel, 250, 150);
    lv_obj_align(ctx.panel, LV_ALIGN_CENTER, 0, 0);

    /* 注册删除前清理回调 */
    lv_obj_add_event_cb(ctx.panel, panel_close_cb, LV_EVENT_DELETE, NULL);

    /* 确认按钮 */
    ctx.confirm_btn = lv_btn_create(ctx.panel);
    lv_obj_align(ctx.confirm_btn, LV_ALIGN_CENTER, 0, 0);
    lv_obj_t * label = lv_label_create(ctx.confirm_btn);
    lv_label_set_text(label, "删除面板");

    lv_obj_add_event_cb(ctx.confirm_btn, confirm_del_cb,
                        LV_EVENT_CLICKED, &ctx);
}

/* 场景：在定时器中安全删除 */
static void timer_delete_cb(lv_timer_t * t)
{
    lv_obj_t * obj = (lv_obj_t *)t->user_data;

    /* 先删除定时器自身 */
    lv_timer_del(t);

    /* 再删除对象（定时器回调中可以直接删除非自身对象） */
    lv_obj_del(obj);
}

void delayed_delete(lv_obj_t * obj, uint32_t delay_ms)
{
    lv_timer_t * timer = lv_timer_create(timer_delete_cb, delay_ms, obj);
    lv_timer_set_repeat_count(timer, 1);  /* 只执行一次 */
}
```

## 附录：i.MX6ULL 平台注意事项

| 项目 | 建议 |
