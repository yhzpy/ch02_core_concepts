## 一、`user_data`：回调函数与应用逻辑的桥梁

### 1.1 核心概念

在 LVGL 中，当你给一个对象注册事件回调时，回调函数内部**无法直接访问你的业务变量**。`user_data` 就是 LVGL 提供的"数据传递通道"——你在注册回调时把数据"塞进去"，回调被触发时再"取出来"。

```
注册时传入 &ctx  ──→  LVGL 内部保存指针  ──→  回调触发时通过 lv_event_get_user_data() 取出
```

### 1.2 典型用法

```c
typedef struct {
    lv_obj_t * label;
    int counter;
} app_context_t;

static app_context_t ctx = { .label = NULL, .counter = 0 };

static void btn_click_cb(lv_event_t * e)
{
    /* 拿出传入的上下文 user_data */
    app_context_t * app = (app_context_t *)lv_event_get_user_data(e);
    app->counter++;
    lv_label_set_text_fmt(app->label, "Count: %d", app->counter);
}

/* 注册时把 &ctx 传进去 */
lv_obj_add_event_cb(btn, btn_click_cb, LV_EVENT_CLICKED, &ctx);
```

### 1.3 不传 `user_data` 会怎样？

**不会崩溃，不会报错**，只是你在回调里取出来的是 `NULL`。

关键看你的回调函数**用不用得到它**：

| 情况 | 传 `NULL` | 传有效指针 |
|------|-----------|-----------|
| 回调里**不需要**外部数据 | ✅ 没问题 | ✅ 没问题，但没必要 |
| 回调里**需要**外部数据 | ❌ 解引用会**段错误崩溃** | ✅ 正确做法 |

**反面例子**（会崩溃）：

```c
/* 注册时传了 NULL */
lv_obj_add_event_cb(btn, on_clicked, LV_EVENT_CLICKED, NULL);

/* 回调里却去解引用 → 💥 段错误 */
static void on_clicked(lv_event_t * e)
{
    button_ctx_t * ctx = (button_ctx_t *)lv_event_get_user_data(e); // ctx = NULL
    ctx->click_count++;  // 💥 对 NULL 解引用 → 崩溃！
}
```

### 1.4 什么时候可以安全地传 `NULL`？

当回调函数**不需要访问任何外部状态**时，可以安全传 `NULL`。例如绘制事件回调，它只关心绘制阶段本身：

```c
static void on_draw(lv_event_t * e)
{
    lv_event_code_t code = lv_event_get_code(e);
    /* 只用到了事件码，不需要任何外部数据 */
    switch (code) {
        case LV_EVENT_DRAW_MAIN_BEGIN: break;
        case LV_EVENT_DRAW_MAIN:       break;
        default:                       break;
    }
}

/* 安全传 NULL */
lv_obj_add_event_cb(btn, on_draw, LV_EVENT_DRAW_ALL, NULL);
```

### 1.5 一句话总结

> **不传 `user_data` 本身完全安全，但如果回调里要用 `lv_event_get_user_data()` 取数据，就必须传一个有效的指针进去，否则就是空指针访问，程序直接崩。**

## 二、Selector：样式生效条件的精确过滤器

### 2.1 核心概念

`selector` 是一个**过滤器**，用来精确指定"这个样式在什么条件下才生效"。它由两部分通过 `|` 按位或组合而成：

```c
selector = Part（部件） | State（状态）
```

| 部分 | 含义 | 常见值 |
|------|------|--------|
| **Part（部件）** | 对象的哪个"部位" | `LV_PART_MAIN`（主体）、`LV_PART_SCROLLBAR`（滚动条）、`LV_PART_KNOB`（旋钮）、`LV_PART_INDICATOR`（指示器）等 |
| **State（状态）** | 对象的什么"状态" | `LV_STATE_DEFAULT`（默认）、`LV_STATE_PRESSED`（按下）、`LV_STATE_CHECKED`（选中）、`LV_STATE_DISABLED`（禁用）等 |

### 2.2 不加 selector 会怎样？

如果不加 selector（即传入 `0`），默认等价于 `LV_PART_MAIN | LV_STATE_DEFAULT`，意思是：

> **主体 + 默认状态 = 无条件生效**

也就是说，不管对象是按下、选中还是禁用，这个样式**始终生效**。

### 2.3 对比示例

```c
/* 不加 selector → 默认状态，始终生效 */
lv_obj_set_style_bg_color(btn, lv_color_red(), 0);
// 0 等价于 LV_PART_MAIN | LV_STATE_DEFAULT
// 按钮无论什么状态，背景都是红色

/* 加了 selector → 只在按下时生效 */
lv_obj_set_style_bg_color(btn, lv_color_blue(),
                          LV_PART_MAIN | LV_STATE_PRESSED);
// 只有按钮被按下时，背景才变蓝色
// 松开后恢复原来的颜色
```

### 2.4 与前端 CSS 的类比

selector 的作用类似于前端 CSS 中的伪类（如 `:hover`、`:active`、`:checked`）：

| LVGL selector | 前端 CSS 类比 | 含义 |
|---------------|---------------|------|
| `LV_STATE_PRESSED` | `:active` | 按下时生效 |
| `LV_STATE_CHECKED` | `:checked` | 选中时生效 |
| `LV_STATE_HOVERED` | `:hover` | 鼠标悬停时生效 |
| `LV_STATE_DISABLED` | `:disabled` | 禁用时生效 |
| `LV_STATE_DEFAULT` | 无伪类（默认） | 始终生效 |

### 2.5 样式应用的完整流程

结合文档中的注释：

```c
/* 是先声明 style 变量，再 set 修饰 style 变量，最后一次性把 style 加到对象身上 */

/* 第1步：声明样式变量（必须 static 或全局） */
static lv_style_t style_btn;

/* 第2步：初始化并设置属性 */
lv_style_init(&style_btn);
lv_style_set_bg_color(&style_btn, lv_color_hex(0x4CAF50));
lv_style_set_radius(&style_btn, 8);

/* 第3步：用 selector 指定生效条件，加到对象上 */
lv_obj_add_style(btn, &style_btn, LV_PART_MAIN | LV_STATE_DEFAULT);
```

### 2.6 一句话总结

> **selector 就是告诉 LVGL："这个样式只在 XX 部位 + XX 状态下才生效"。不加 selector 就是"主体 + 默认状态"，也就是无条件生效。**

## 四、关键要点速记

1. **`user_data` 传不传？**
   - 回调需要外部数据 → **必须传有效指针**，否则段错误
   - 回调不需要外部数据 → 传 `NULL` 即可

2. **selector 加不加？**
   - 需要区分状态（按下/选中/禁用等）→ **加 selector**，让不同状态有不同外观
   - 样式始终生效、不区分状态 → **不加**（默认 `LV_PART_MAIN | LV_STATE_DEFAULT`）

3. **样式三步骤：**
   - **声明** → `static lv_style_t style;`
   - **修饰** → `lv_style_set_xxx(&style, ...);`
   - **添加** → `lv_obj_add_style(obj, &style, selector);`
