## 2.1 对象树模型 (Object Tree) {#21-对象树模型-object-tree}

LVGL 的 UI 由一棵**对象树**构成。每一个可见或不可见的元素——按钮、标签、滑块、甚至一个空白容器——都是一个 `lv_obj_t` 实例。理解这棵树的结构是掌握 LVGL 的第一步。

### 2.1.1 lv_obj_t —— 万物皆对象

在 LVGL 中，`lv_obj_t` 是所有 UI 元素的**基类**（base object）。无论是一个简单的矩形容器还是复杂的图表控件，它们的底层都是 `lv_obj_t`。LVGL 使用 C 语言实现了面向对象的继承体系：

```c
/* lvgl/src/core/lv_obj.h 中的简化结构 */
typedef struct _lv_obj_t {
    const lv_obj_class_t * class_p;   /* 指向类描述符 */
    struct _lv_obj_t * parent;        /* 父对象指针 */
    lv_coord_t coords_x1, coords_y1;  /* 坐标 */
    lv_coord_t coords_x2, coords_y2;
    lv_area_t coords;                 /* 完整区域 */
    lv_style_list_t style_list;       /* 样式链表 */
    lv_ll_t spec_attr;                /* 特殊属性链表（事件、子对象等） */
    lv_obj_flag_t flags;              /* 对象标志位 */
    /* ... */
} lv_obj_t;
```

每个 widget 类（如 `lv_btn_class`, `lv_label_class`）都继承自 `lv_obj_class`，并在其基础上扩展属性和行为。当你调用 `lv_btn_create()` 时，LVGL 内部会：

1. 分配 `lv_obj_t` 的内存（实际大小由 btn 类决定）
2. 设置 `class_p` 指向 `lv_btn_class`
3. 将新对象加入父对象的子对象链表

### 2.1.2 父子关系与对象创建

LVGL 使用 `lv_obj_create(parent)` 创建基础对象，所有 widget 创建函数都遵循相同的签名模式：

```c
/* 创建基础对象（空白容器） */
lv_obj_t * parent = lv_obj_create(NULL);   /* parent = NULL → 创建在默认屏幕上 */

/* 在 parent 上创建子对象 */
lv_obj_t * child1 = lv_obj_create(parent);
lv_obj_t * child2 = lv_obj_create(parent);

/* 在 child1 上再创建孙对象 */
lv_obj_t * grandchild = lv_label_create(child1);
lv_label_set_text(grandchild, "Hello LVGL!");
```

**关键规则**：

- 子对象的位置是**相对于父对象**的（相对坐标系）
- 子对象的可见区域不会超出父对象的边界（裁剪）
- 删除父对象会**自动递归删除**所有子对象

屏幕层级结构如下：

```
Screen (lv_scr_act())
├── Container A
│   ├── Button 1
│   │   └── Label "OK"
│   └── Button 2
│       └── Label "Cancel"
├── Container B
│   └── Slider
└── Label "Title"
```

### 2.1.3 对象的销毁

LVGL 提供了三种销毁对象的方式：

```c
/* 1. 立即删除对象及其所有子对象 */
lv_obj_del(obj);

/* 2. 异步删除 —— 在当前事件处理完成后删除 */
lv_obj_del_async(obj);

/* 3. 清理对象的所有子对象，但保留对象本身 */
lv_obj_clean(obj);
```

**使用场景对比**：

| 函数 | 行为 | 适用场景 |
|------|------|----------|
| `lv_obj_del(obj)` | 立即删除 obj 及其子树 | 确定不再需要该对象 |
| `lv_obj_del_async(obj)` | 延迟到事件处理完毕后删除 | 在事件回调中删除触发事件的对象自身 |
| `lv_obj_clean(obj)` | 删除所有子对象，保留 obj | 清空容器内容后重新填充 |

> **警告**：在事件回调中**绝不要**使用 `lv_obj_del()` 删除正在处理事件的对象本身，这会导致 use-after-free。应使用 `lv_obj_del_async()`。

### 2.1.4 对象遍历

LVGL 提供 API 来遍历对象树的子节点：

```c
/* 获取子对象数量 */
uint32_t cnt = lv_obj_get_child_cnt(parent);

/* 按索引获取子对象（0-based） */
for (uint32_t i = 0; i < cnt; i++) {
    lv_obj_t * child = lv_obj_get_child(parent, i);
    printf("Child %u: %p\n", i, (void *)child);
}

/* 获取父对象 */
lv_obj_t * p = lv_obj_get_parent(child);

/* 获取对象在父对象中的索引 */
uint32_t idx = lv_obj_get_index(child);
```

### 2.1.5 对象标志 (Flags)

`lv_obj_t` 的 `flags` 字段是一个位掩码，控制对象的行为特性：

```c
/* 设置标志 *//*对象的行为特性*/
lv_obj_add_flag(obj, LV_OBJ_FLAG_CLICKABLE);      /* 可点击 */
lv_obj_add_flag(obj, LV_OBJ_FLAG_SCROLLABLE);      /* 可滚动 */
lv_obj_add_flag(obj, LV_OBJ_FLAG_HIDDEN);           /* 隐藏 */
lv_obj_add_flag(obj, LV_OBJ_FLAG_CLICK_FOCUSABLE); /* 可被点击聚焦 */

/* 清除标志 */
lv_obj_clear_flag(obj, LV_OBJ_FLAG_SCROLLABLE);    /* 禁止滚动 */
lv_obj_clear_flag(obj, LV_OBJ_FLAG_HIDDEN);         /* 取消隐藏 */

/* 检查标志 */
bool hidden = lv_obj_has_flag(obj, LV_OBJ_FLAG_HIDDEN);
bool clickable = lv_obj_has_flag(obj, LV_OBJ_FLAG_CLICKABLE);
```

**常用标志一览**：

| 标志 | 含义 |
|------|------|
| `LV_OBJ_FLAG_HIDDEN` | 隐藏对象及其子对象，不参与渲染和输入 |
| `LV_OBJ_FLAG_CLICKABLE` | 接受点击事件 |
| `LV_OBJ_FLAG_CLICK_FOCUSABLE` | 可通过点击获得焦点 |
| `LV_OBJ_FLAG_SCROLLABLE` | 内容超出时可滚动 |
| `LV_OBJ_FLAG_SCROLL_ELASTIC` | 滚动到边界时有弹性效果 |
| `LV_OBJ_FLAG_SCROLL_MOMENTUM` | 惯性滚动 |
| `LV_OBJ_FLAG_SCROLL_ONE` | 每次滚动一个子对象 |
| `LV_OBJ_FLAG_SNAPPABLE` | 滚动时自动对齐到子对象 |
| `LV_OBJ_FLAG_PRESS_LOCK` | 按下时锁定父对象的滚动 |
| `LV_OBJ_FLAG_EVENT_BUBBLE` | 事件冒泡到父对象 |
| `LV_OBJ_FLAG_GESTURE_BUBBLE` | 手势冒泡到父对象 |
| `LV_OBJ_FLAG_ADV_HITTEST` | 高级命中测试（考虑圆角等） |
| `LV_OBJ_FLAG_IGNORE_LAYOUT` | 忽略布局（Layout） |
| `LV_OBJ_FLAG_FLOATING` | 浮动对象（不影响父对象的布局计算） |
| `LV_OBJ_FLAG_OVERFLOW_VISIBLE` | 内容溢出父边界时仍然可见 |
| `LV_OBJ_FLAG_USER_1` ~ `LV_OBJ_FLAG_USER_4` | 用户自定义标志 |

### 2.1.6 屏幕对象

LVGL 维护着特殊的屏幕层级结构：

```c
/* 当前活动屏幕 —— 所有普通 widget 的根 */
lv_obj_t * scr = lv_scr_act();

/* 切换屏幕（会销毁旧屏幕） */
lv_obj_t * new_scr = lv_obj_create(NULL);
lv_scr_load(new_scr);

/* 带动画的屏幕切换 */
lv_scr_load_anim(new_scr, LV_SCR_LOAD_ANIM_FADE_ON, 500, 0, false);
/* 参数：新屏幕, 动画类型, 持续时间ms, 延迟ms, 是否自动删除旧屏幕 */

/* 顶层覆盖层 —— 叠加在所有屏幕之上，常用于弹窗 */
lv_obj_t * top = lv_layer_top();

/* 系统覆盖层 —— 最高层级，用于系统级提示 */
lv_obj_t * sys = lv_layer_sys();
```

层级关系：

```
lv_layer_sys()       ← 最顶层（系统层）
  └── ...
lv_layer_top()       ← 覆盖层（弹窗、Toast）
  └── ...
lv_scr_act()         ← 当前活动屏幕（主 UI）
  └── ...
```

### 2.1.7 代码示例：构建 Widget 层级

```c
/**
 * 示例：构建一个简单的控制面板界面
 * 演示父子关系、对象遍历和标志设置
 */
#include "lvgl.h"

static void build_control_panel(void)
{
    /* 获取当前屏幕作为根 */
    lv_obj_t * scr = lv_scr_act();

    /* 创建标题标签 */
    lv_obj_t * title = lv_label_create(scr);
    lv_label_set_text(title, "Control Panel");
    lv_obj_align(title, LV_ALIGN_TOP_MID, 0, 20);

    /* 创建按钮容器 */
    lv_obj_t * btn_container = lv_obj_create(scr);
    lv_obj_set_size(btn_container, 300, 200);
    lv_obj_center(btn_container);
    lv_obj_set_flex_flow(btn_container, LV_FLEX_FLOW_ROW_WRAP);
    lv_obj_set_flex_align(btn_container, LV_FLEX_ALIGN_CENTER,
                          LV_FLEX_ALIGN_CENTER, LV_FLEX_ALIGN_CENTER);

    /* 清除容器的默认滚动和点击 */
    lv_obj_clear_flag(btn_container, LV_OBJ_FLAG_SCROLLABLE);

    /* 在容器中创建多个按钮 */
    const char * btn_texts[] = {"Start", "Stop", "Reset", "Config"};
    for (int i = 0; i < 4; i++) {
        lv_obj_t * btn = lv_btn_create(btn_container);
        lv_obj_set_size(btn, 120, 50);

        lv_obj_t * label = lv_label_create(btn);
        lv_label_set_text(label, btn_texts[i]);
        lv_obj_center(label);
    }

    /* 遍历并打印所有按钮的信息 */
    uint32_t cnt = lv_obj_get_child_cnt(btn_container);/* 获取子对象数量 */
    printf("Container has %u children\n", cnt);
    for (uint32_t i = 0; i < cnt; i++) {
        lv_obj_t * child = lv_obj_get_child(btn_container, i);
        /* 每个 child 是一个 button，其子对象是 label */
        lv_obj_t * btn_label = lv_obj_get_child(child, 0);
        if (btn_label) {
            printf("  Button %u: '%s'\n", i, lv_label_get_text(btn_label));
        }
    }

    /* 动态删除第二个按钮（Stop） */
    lv_obj_t * stop_btn = lv_obj_get_child(btn_container, 1);/* 获取第 2 个子对象 */
    lv_obj_del(stop_btn);

    /* 或者用 lv_obj_clean 清空所有按钮 */
    // lv_obj_clean(btn_container);
}
```

## 2.2 事件机制 (Event System) {#22-事件机制-event-system}

事件系统是 LVGL 交互的核心。用户的每一次点击、滑动、键盘输入，以及系统内部的绘制请求、值变化等，都会产生事件。

### 2.2.1 事件回调注册

使用 `lv_obj_add_event_cb()` 为对象注册事件处理函数：

```c
/* 函数签名 */
lv_event_dsc_t * lv_obj_add_event_cb(lv_obj_t * obj,
                                      lv_event_cb_t event_cb,
                                      lv_event_code_t filter,
                                      void * user_data);

/* event_cb 类型定义 */
typedef void (*lv_event_cb_t)(lv_event_t * e);
```

参数说明：

- `obj`：要监听事件的对象
- `event_cb`：回调函数指针
- `filter`：事件码过滤器，设为 `LV_EVENT_ALL` 接收所有事件
- `user_data`：用户自定义数据指针，可在回调中获取

回调函数返回一个 `lv_event_dsc_t *`，可用于后续移除该回调：

```c
lv_event_dsc_t * dsc = lv_obj_add_event_cb(btn, my_cb, LV_EVENT_CLICKED, NULL);

/* 后续移除 */
lv_obj_remove_event_dsc(btn, dsc);
```

### 2.2.2 事件码 (Event Code)

事件码是一个 `lv_event_code_t` 枚举值，表示事件的类型：

**用户交互事件**：

| 事件码 | 含义 |
|--------|------|
| `LV_EVENT_PRESSED` | 对象被按下 |
| `LV_EVENT_PRESSING` | 持续按压中 |
| `LV_EVENT_PRESS_LOST` | 按压丢失（手指滑出） |
| `LV_EVENT_SHORT_CLICKED` | 短点击 |
| `LV_EVENT_LONG_PRESSED` | 长按 |
| `LV_EVENT_LONG_PRESSED_REPEAT` | 长按重复触发 |
| `LV_EVENT_CLICKED` | 点击完成（按下+释放） |
| `LV_EVENT_RELEASED` | 释放 |
| `LV_EVENT_SCROLL_BEGIN` | 滚动开始 |
| `LV_EVENT_SCROLL_END` | 滚动结束 |
| `LV_EVENT_SCROLL` | 滚动中 |
| `LV_EVENT_GESTURE` | 手势识别 |
| `LV_EVENT_KEY` | 按键输入 |
| `LV_EVENT_FOCUSED` | 获得焦点 |
| `LV_EVENT_DEFOCUSED` | 失去焦点 |

**值变化事件**：

| 事件码 | 含义 |
|--------|------|
| `LV_EVENT_VALUE_CHANGED` | 值改变（如滑块、开关、下拉框） |
| `LV_EVENT_REFRESH` | 刷新请求 |

**绘制事件**：

| 事件码 | 含义 |
|--------|------|
| `LV_EVENT_DRAW_MAIN_BEGIN` | 主绘制阶段开始 |
| `LV_EVENT_DRAW_MAIN` | 主绘制阶段 |
| `LV_EVENT_DRAW_MAIN_END` | 主绘制阶段结束 |
| `LV_EVENT_DRAW_POST_BEGIN` | 后绘制阶段开始 |
| `LV_EVENT_DRAW_POST` | 后绘制阶段（覆盖在子对象之上） |
| `LV_EVENT_DRAW_POST_END` | 后绘制阶段结束 |
| `LV_EVENT_DRAW_PART_BEGIN` | 局部绘制开始 |
| `LV_EVENT_DRAW_PART_END` | 局部绘制结束 |

**生命周期事件**：

| 事件码 | 含义 |
|--------|------|
| `LV_EVENT_CREATE` | 对象创建完成 |
| `LV_EVENT_DELETE` | 对象即将删除 |
| `LV_EVENT_CHILD_CREATED` | 子对象创建 |
| `LV_EVENT_CHILD_DELETED` | 子对象删除 |
| `LV_EVENT_SIZE_CHANGED` | 尺寸变化 |
| `LV_EVENT_STYLE_CHANGED` | 样式变化 |
| `LV_EVENT_LAYOUT_CHANGED` | 布局变化 |

### 2.2.3 事件数据结构

在回调函数中，通过 `lv_event_t` 结构获取事件详细信息：

```c
static void my_event_cb(lv_event_t * e)
{
    /* 获取事件码 */
    lv_event_code_t code = lv_event_get_code(e);

    /* 获取触发事件的目标对象（最底层的事件源） */
    lv_obj_t * target = lv_event_get_target(e);

    /* 获取当前对象（注册了此回调的对象，可能与 target 不同） */
    lv_obj_t * current = lv_event_get_current_target(e);

    /* 获取用户数据 */
    void * user_data = lv_event_get_user_data(e);

    switch (code) {
        case LV_EVENT_CLICKED:
            printf("Object clicked!\n");
            break;
        case LV_EVENT_VALUE_CHANGED:
            printf("Value changed!\n");
            break;
        default:
            break;
    }
}
```

**target vs current_target**：当启用事件冒泡时，两者不同：

- `target`：实际触发事件的对象（如按钮中的 label）
- `current_target`：当前正在处理事件回调的对象（如按钮本身）

### 2.2.4 事件冒泡与阻止

默认情况下，事件**不会冒泡**到父对象。要启用冒泡，需设置 `LV_OBJ_FLAG_EVENT_BUBBLE`：

```c
/* 启用事件冒泡 */
lv_obj_add_flag(child_obj, LV_OBJ_FLAG_EVENT_BUBBLE);

/* 在回调中阻止事件继续冒泡 */
static void child_event_cb(lv_event_t * e)
{
    lv_event_stop_bubbling(e);   /* 阻止事件向父对象传播 */
    printf("Child event handled, bubbling stopped.\n");
}
```

事件冒泡的传播路径：child → parent → grandparent → ... → screen

某些事件不会冒泡，包括绘制事件和生命周期事件。

### 2.2.5 自定义事件发送

你可以手动发送事件：

```c
/* 发送一个标准事件 */
lv_res_t res = lv_event_send(obj, LV_EVENT_CLICKED, NULL);

/* 发送带自定义数据的事件 */
int my_data = 42;
lv_event_send(obj, LV_EVENT_REFRESH, &my_data);

/* 在回调中获取自定义数据 */
static void on_refresh(lv_event_t * e)
{
    int * data = (int *)lv_event_get_param(e);
    if (data) {
        printf("Refresh with data: %d\n", *data);
    }
}
```

### 2.2.6 用户数据传递

`user_data` 是连接 LVGL 回调与你的应用逻辑的桥梁：

```c
/* 方式一：直接传递指针 */
typedef struct {
    lv_obj_t * label;
    int counter;
} app_context_t;

static app_context_t ctx = { .label = NULL, .counter = 0 };

static void btn_click_cb(lv_event_t * e)
{
    app_context_t * app = (app_context_t *)lv_event_get_user_data(e);
    app->counter++;
    lv_label_set_text_fmt(app->label, "Count: %d", app->counter);
}

void create_counter_ui(void)
{
    lv_obj_t * scr = lv_scr_act();

    ctx.label = lv_label_create(scr);
    lv_label_set_text(ctx.label, "Count: 0");
    lv_obj_align(ctx.label, LV_ALIGN_CENTER, 0, -40);

    lv_obj_t * btn = lv_btn_create(scr);
    lv_obj_align(btn, LV_ALIGN_CENTER, 0, 40);
    lv_obj_add_event_cb(btn, btn_click_cb, LV_EVENT_CLICKED, &ctx);

    lv_obj_t * btn_label = lv_label_create(btn);
    lv_label_set_text(btn_label, "Increment");
}
```

### 2.2.7 代码示例：多事件处理器与事件流

```c
/**
 * 示例：一个按钮注册多个事件处理器
 * 演示事件码过滤、事件流控制和 user_data 的使用
 */
#include "lvgl.h"
#include <stdio.h>

typedef struct {
    lv_obj_t * status_label;
    uint32_t click_count;
} button_ctx_t;

static button_ctx_t btn_ctx;

/* 处理器1：只处理 LV_EVENT_CLICKED */
static void on_clicked(lv_event_t * e)
{
    button_ctx_t * ctx = (button_ctx_t *)lv_event_get_user_data(e);/* 拿出传入的上下文 user_data */
    ctx->click_count++;
    lv_label_set_text_fmt(ctx->status_label,
                          "Clicked %u times", ctx->click_count);
    printf("[on_clicked] click #%u\n", ctx->click_count);
}

/* 处理器2：处理长按事件 */
static void on_long_press(lv_event_t * e)
{
    printf("[on_long_press] Button long pressed!\n");
    /* 长按重置计数器 */
    button_ctx_t * ctx = (button_ctx_t *)lv_event_get_user_data(e);
    ctx->click_count = 0;
    lv_label_set_text(ctx->status_label, "Counter reset!");
}

/* 处理器3：处理所有绘制事件 */
static void on_draw(lv_event_t * e)
{
    lv_event_code_t code = lv_event_get_code(e);
    switch (code) {
        case LV_EVENT_DRAW_MAIN_BEGIN:
            /* 可以在这里修改绘制参数 */
            break;
        case LV_EVENT_DRAW_MAIN:
            /* 主绘制阶段 */
            break;
        default:
            break;
    }
}

/* 处理器4：值变化（用于带 checkable 的按钮） */
static void on_value_changed(lv_event_t * e)
{
    lv_obj_t * btn = lv_event_get_target(e);
    bool checked = lv_obj_has_state(btn, LV_STATE_CHECKED);
    printf("[on_value_changed] Button is now %s\n",
           checked ? "CHECKED" : "UNCHECKED");
}

void create_multi_event_demo(void)
{
    lv_obj_t * scr = lv_scr_act();
    btn_ctx.click_count = 0;

    /* 创建状态标签 */
    btn_ctx.status_label = lv_label_create(scr);
    lv_label_set_text(btn_ctx.status_label, "Clicked 0 times");
    lv_obj_align(btn_ctx.status_label, LV_ALIGN_CENTER, 0, -60);

    /* 创建按钮 */
    lv_obj_t * btn = lv_btn_create(scr);
    lv_obj_set_size(btn, 160, 60);
    lv_obj_align(btn, LV_ALIGN_CENTER, 0, 20);
    lv_obj_add_flag(btn, LV_OBJ_FLAG_CLICKABLE);

    /* 启用 checkable 以触发 VALUE_CHANGED */
    lv_obj_add_flag(btn, LV_OBJ_FLAG_CHECKABLE);

    /* 注册多个事件处理器 */
    lv_obj_add_event_cb(btn, on_clicked,     LV_EVENT_CLICKED,        &btn_ctx);
    lv_obj_add_event_cb(btn, on_long_press,  LV_EVENT_LONG_PRESSED,   &btn_ctx);
    lv_obj_add_event_cb(btn, on_draw,        LV_EVENT_DRAW_ALL,       NULL);
    lv_obj_add_event_cb(btn, on_value_changed, LV_EVENT_VALUE_CHANGED, &btn_ctx);

    lv_obj_t * label = lv_label_create(btn);
    lv_label_set_text(label, "Click / Hold");
    lv_obj_center(label);
}
```

## 2.3 样式系统 (Style System) {#23-样式系统-style-system}

样式系统是 LVGL 最强大的特性之一，它让你像写 CSS 一样控制 UI 的外观。样式与对象分离，可以复用于多个对象。

### 2.3.1 lv_style_t 结构

样式是一个独立的值集合，每个样式可以包含多个属性：

```c
/* 声明样式变量（必须为 static 或全局，因为 LVGL 只保存指针） */
static lv_style_t style_btn;

/* 初始化样式 */
lv_style_init(&style_btn);

/* 设置各种属性 */
lv_style_set_width(&style_btn, 120);
lv_style_set_height(&style_btn, 50);
lv_style_set_bg_color(&style_btn, lv_color_hex(0x4CAF50));
lv_style_set_bg_opa(&style_btn, LV_OPA_COVER);
lv_style_set_border_width(&style_btn, 2);
lv_style_set_border_color(&style_btn, lv_color_hex(0x2E7D32));
lv_style_set_radius(&style_btn, 8);
lv_style_set_pad_all(&style_btn, 10);
lv_style_set_text_color(&style_btn, lv_color_white());
```

> **重要**：样式变量必须是 `static` 或全局的！LVGL 内部只存储指针，如果样式是局部变量，函数返回后内存就会被回收，导致未定义行为。

### 2.3.2 样式属性分类

### 2.3.3 样式添加与 Selector

将样式应用到对象时使用 `lv_obj_add_style()`：

```c
lv_obj_add_style(obj, &style, selector);/* 是先声明style变量 再set修饰style变量  最后一次性把style加到对象身上*/
```

`selector` 是 **part（部件）** 和 **state（状态）** 的按位或组合：

```c
/* selector = part | state */
lv_obj_add_style(btn, &style_default, LV_PART_MAIN | LV_STATE_DEFAULT);
lv_obj_add_style(btn, &style_pressed, LV_PART_MAIN | LV_STATE_PRESSED);
lv_obj_add_style(slider, &style_knob, LV_PART_KNOB | LV_STATE_DEFAULT);
lv_obj_add_style(slider, &style_ind,  LV_PART_INDICATOR | LV_STATE_DEFAULT);
```

如果只指定 part 不指定 state，LVGL 会默认使用 `LV_STATE_DEFAULT`；反过来也可以：

```c
/* 以下等价 */
lv_obj_add_style(obj, &s, LV_PART_MAIN | LV_STATE_DEFAULT);
lv_obj_add_style(obj, &s, LV_PART_MAIN);
lv_obj_add_style(obj, &s, LV_STATE_DEFAULT);
lv_obj_add_style(obj, &s, 0);  /* 默认就是 MAIN + DEFAULT */
```

### 2.3.4 Parts（部件）

LVGL 的 widget 由多个部件组成，不同部件可以应用不同样式：

| Part 常量 | 含义 | 典型使用 |
|-----------|------|----------|
| `LV_PART_MAIN` | 主部件 | 几乎所有对象 |
| `LV_PART_SCROLLBAR` | 滚动条 | 可滚动容器 |
| `LV_PART_INDICATOR` | 指示器 | 滑块、进度条、仪表 |
| `LV_PART_KNOB` | 旋钮 | 滑块、开关 |
| `LV_PART_SELECTED` | 选中部分 | 下拉框、图表 |
| `LV_PART_ITEMS` | 列表项 | 列表、表格、菜单 |
| `LV_PART_TICKS` | 刻度 | 仪表 |
| `LV_PART_CURSOR` | 光标 | 文本区域 |
| `LV_PART_ANY` | 任意部件 | 用于查询 |

以 `lv_slider_t` 为例，它有三个 part：

```
滑块结构：
┌──────────────────────────────┐
│ MAIN（背景轨道）              │
│   ┌────────────────┐         │
│   │ INDICATOR（填充部分） │   │
│   │        ┌──┐    │         │
│   │        │KN│    │         │
│   │        │OB│    │         │
│   │        └──┘    │         │
│   └────────────────┘         │
└──────────────────────────────┘
```

### 2.3.5 States（状态）

LVGL 对象可以同时处于多个状态，状态之间可以组合：

| State 常量 | 含义 |
|------------|------|
| `LV_STATE_DEFAULT` | 默认状态（值为 0x0000） |
| `LV_STATE_CHECKED` | 被选中/勾选 |
| `LV_STATE_FOCUSED` | 获得焦点 |
| `LV_STATE_FOCUS_KEY` | 通过键盘获得焦点 |
| `LV_STATE_EDITED` | 正在编辑（如文本框） |
| `LV_STATE_HOVERED` | 鼠标悬停 |
| `LV_STATE_PRESSED` | 被按下 |
| `LV_STATE_SCROLLED` | 正在滚动 |
| `LV_STATE_DISABLED` | 禁用 |
| `LV_STATE_USER_1` ~ `LV_STATE_USER_4` | 用户自定义状态 |
| `LV_STATE_ANY` | 任意状态（用于查询） |

**状态优先级**：当多个状态的样式冲突时，优先级从低到高为：

```
DEFAULT < CHECKED < FOCUSED < FOCUS_KEY < EDITED < HOVERED < PRESSED < SCROLLED < DISABLED
```

更高优先级的状态样式会覆盖低优先级的。

### 2.3.6 本地样式 vs 全局样式

```c
/* 全局样式 —— 可复用于多个对象 */
static lv_style_t style_btn_green;
lv_style_init(&style_btn_green);
lv_style_set_bg_color(&style_btn_green, lv_color_hex(0x4CAF50));

/* 应用到多个对象 */
lv_obj_add_style(btn1, &style_btn_green, 0);
lv_obj_add_style(btn2, &style_btn_green, 0);
lv_obj_add_style(btn3, &style_btn_green, 0);

/* 本地样式 —— 只属于一个对象的快速设置 */
/* 方式一：通过 lv_obj_set_* 系列函数直接设置 */
lv_obj_set_style_bg_color(btn4, lv_color_hex(0xFF5722), 0);
lv_obj_set_style_bg_opa(btn4, LV_OPA_COVER, 0);
lv_obj_set_style_radius(btn4, 12, 0);

/* 方式二：带 selector 的本地样式 */
lv_obj_set_style_bg_color(btn5, lv_color_hex(0x2196F3),
                          LV_PART_MAIN | LV_STATE_PRESSED);
```
/* selector就相当于前端的hover 表示只有当按下或是其他什么条件的时候才会添加这个样式 */
**选择建议**：

- 多个对象共享的外观 → 全局样式（`lv_style_t`）
- 单个对象的特殊调整 → 本地样式（`lv_obj_set_style_*`）
- 性能敏感场景 → 全局样式（只需一份样式数据）

### 2.3.7 样式过渡 (Transition)

LVGL 支持属性值的动画过渡，通过 `lv_style_transition_dsc_t` 配置：

```c
/* 定义需要过渡的属性 */
static const lv_style_prop_t trans_props[] = {
    LV_STYLE_BG_COLOR,
    LV_STYLE_BG_OPA,
    LV_STYLE_BORDER_WIDTH,
    LV_STYLE_PROP_INV  /* 数组结束标记 */
};

/* 创建过渡描述符 */  /*过渡描述符->style->obj */
static lv_style_transition_dsc_t trans;
lv_style_transition_dsc_init(&trans);
lv_style_transition_dsc_set(&trans, trans_props,
                            lv_anim_path_ease_in_out,
                            300,   /* 持续时间 ms */
                            0,     /* 延迟 ms */
                            NULL); /* user_data */

/* 将过渡应用到样式 */
static lv_style_t style_pressed;
lv_style_init(&style_pressed);
lv_style_set_transition(&style_pressed, &trans);
lv_style_set_bg_color(&style_pressed, lv_color_hex(0x1565C0));

/* 应用到按钮的按下状态 */
lv_obj_add_style(btn, &style_pressed, LV_PART_MAIN | LV_STATE_PRESSED);
```

> **注意**：过渡描述符（`trans`）和属性数组（`trans_props`）也必须是 `static` 或全局变量。

## 2.4 对象生命周期 {#24-对象生命周期}

理解 LVGL 对象的完整生命周期有助于避免内存泄漏和野指针问题。

### 2.4.1 生命周期阶段

```
  创建 (Create)
    │
    ▼
  配置 (Configure)
    │  ├── 设置尺寸、位置
    │  ├── 设置属性（文本、范围等）
    │  └── 添加标志
    │
    ▼
  添加样式 (Style)
    │  ├── lv_obj_add_style()
    │  └── lv_obj_set_style_*()
    │
    ▼
  注册事件 (Event)
    │  └── lv_obj_add_event_cb()
    │
    ▼
  使用 (Runtime)
    │  ├── 响应用户交互
    │  ├── 动态修改属性
    │  └── 触发重绘
    │
    ▼
  销毁 (Destroy)
       ├── lv_obj_del()
       ├── lv_obj_del_async()
       └── lv_obj_clean()
```

### 2.4.2 内存管理注意事项

1. **样式必须持久**：`lv_style_t` 变量必须是 `static` 或全局的。LVGL 只保存指针引用，不复制样式数据。

2. **事件回调中不要删除自身**：

```c
/* 错误！在回调中删除正在处理事件的对象 */
static void bad_cb(lv_event_t * e)
{
    lv_obj_t * btn = lv_event_get_target(e);
    lv_obj_del(btn);  /* 危险！可能导致 use-after-free */
}

/* 正确：使用 lv_obj_del_async *//* 异步删除，事件处理完毕再删除 */
static void good_cb(lv_event_t * e)
{
    lv_obj_t * btn = lv_event_get_target(e);
    lv_obj_del_async(btn);  /* 安全：事件处理完毕后再删除 */
}
```

3. **字符串生命周期**：

```c
/* lv_label_set_text() 会复制字符串，安全 */
void create_label(void) {
    char buf[32];
    snprintf(buf, sizeof(buf), "Hello %d", 42);
    lv_label_set_text(label, buf);  /* buf 被复制，函数返回后安全 */
}

/* lv_label_set_text_static() 不复制，调用者负责生命周期 */
static const char * persistent_str = "Hello";
lv_label_set_text_static(label, persistent_str);  /* 必须保证 persistent_str 一直有效 */
```

4. **屏幕切换时旧屏幕的销毁**：

```c
/* auto_del = true：自动删除旧屏幕 */
lv_scr_load_anim(new_scr, LV_SCR_LOAD_ANIM_MOVE_LEFT, 500, 0, true);

/* auto_del = false：保留旧屏幕，可以后续手动删除或切回 */
lv_scr_load_anim(new_scr, LV_SCR_LOAD_ANIM_MOVE_LEFT, 500, 0, false);
```

### 2.4.3 三种删除方式的内部机制

```c
/* lv_obj_del()：立即递归删除 */

{
```

## 2.5 主循环与 Tick 机制 {#25-主循环与-tick-机制}

LVGL 不是独立运行的框架，而是**被你的主循环驱动**的库。理解这个机制对于在 i.MX6ULL 上正确集成 LVGL 至关重要。

### 2.5.1 lv_tick_inc() —— 时间基准

LVGL 需要一个精确的毫秒级时钟来驱动动画、定时器和输入处理：

```c
/* 必须定期调用，告知 LVGL 经过了多少毫秒 */
void lv_tick_inc(uint32_t tick_period);
```

在 i.MX6ULL Linux 环境下，有几种提供 tick 的方式：

```c
/* 方式一：在主循环中调用 */
while (running) {
    uint32_t time_till_next = lv_timer_handler();/*返回到下次调用建议等待的毫秒数 */
    usleep(time_till_next * 1000);
    lv_tick_inc(time_till_next);
}

/* 方式二：使用定时器中断（推荐） */
#include <signal.h>
#include <sys/timerfd.h>

static void timer_handler(int sig)
{
    lv_tick_inc(5);  /* 每 5ms tick 一次 */
}

/* 在主函数中设置 5ms 定时器 */
struct sigaction sa;
sa.sa_handler = timer_handler;
sigemptyset(&sa.sa_mask);
sa.sa_flags = SA_RESTART;
sigaction(SIGALRM, &sa, NULL);

struct itimerval timer;
timer.it_interval.tv_sec = 0;
timer.it_interval.tv_usec = 5000;  /* 5ms */
timer.it_value = timer.it_interval;
setitimer(ITIMER_REAL, &timer, NULL);
```

### 2.5.2 lv_timer_handler() —— 任务调度核心

`lv_timer_handler()` 是 LVGL 的"心脏"，它负责：

1. 处理到期的定时器（`lv_timer_t`）
2. 执行动画
3. 刷新显示（重绘脏区域）
4. 处理输入设备事件
5. 执行异步删除

```c
/* 返回值：到下一次调用建议等待的毫秒数 */
uint32_t time_till_next = lv_timer_handler();
```

**调用频率建议**：

- 普通应用：每 5~10ms 调用一次
- 动画密集：每 1~5ms 调用一次
- 简单静态界面：每 20~50ms 调用一次

### 2.5.3 lv_timer 的创建与管理

`lv_timer` 是 LVGL 内置的定时器机制，用于周期性执行回调：

```c
/* 创建定时器 */
lv_timer_t * timer = lv_timer_create(my_timer_cb, period_ms, user_data);

/* 定时器回调签名 */
static void my_timer_cb(lv_timer_t * timer)
{
    void * user_data = timer->user_data;
    /* 周期性任务 */
}

/* 管理定时器 */
lv_timer_set_period(timer, 1000);     /* 修改周期 */
lv_timer_set_repeat_count(timer, 10); /* 设置重复次数（-1=无限，0=立即删除） */
lv_timer_pause(timer);                 /* 暂停 */
lv_timer_resume(timer);                /* 恢复 */
lv_timer_del(timer);                   /* 删除 */
lv_timer_reset(timer);                 /* 重置计时 */

/* 一次性定时器 */
lv_timer_t * once = lv_timer_create(once_cb, 2000, NULL);
lv_timer_set_repeat_count(once, 1);  /* 只执行一次后自动删除 *//*设置重复次数*/
```

### 2.5.4 与 Linux epoll/select 的集成方案

在 i.MX6ULL 嵌入式 Linux 中，主循环通常需要同时处理多种事件源（输入设备、网络、文件描述符等）。将 LVGL 与 `epoll` 集成是一种高效方案：

```c
/**
 * 示例：将 LVGL 主循环与 epoll 集成
 * 同时处理触摸屏输入、网络和 LVGL 定时器
 */
#include "lvgl.h"
#include <sys/epoll.h>
#include <sys/timerfd.h>
#include <fcntl.h>
#include <unistd.h>
#include <stdio.h>

#define MAX_EVENTS 16
#define LVGL_PERIOD_MS 5

static int setup_timer_fd(int period_ms)
{
    int tfd = timerfd_create(CLOCK_MONOTONIC, TFD_NONBLOCK);
    if (tfd < 0) return -1;

    struct itimerspec its;
    its.it_interval.tv_sec  = period_ms / 1000;
    its.it_interval.tv_nsec = (period_ms % 1000) * 1000000L;
    its.it_value = its.it_interval;

    timerfd_settime(tfd, 0, &its, NULL);
    return tfd;
}

int main_loop_with_epoll(int touch_fd, int network_fd)
{
    int epfd = epoll_create1(0);
    if (epfd < 0) {
        perror("epoll_create1");
        return -1;
    }

    /* 创建 LVGL 定时器文件描述符 */
    int lvgl_tfd = setup_timer_fd(LVGL_PERIOD_MS);
    if (lvgl_tfd < 0) {
        perror("timerfd_create");
        close(epfd);
        return -1;
    }

    /* 注册文件描述符到 epoll */
    struct epoll_event ev;

    ev.events = EPOLLIN;
    ev.data.fd = touch_fd;
    epoll_ctl(epfd, EPOLL_CTL_ADD, touch_fd, &ev);

    ev.events = EPOLLIN;
    ev.data.fd = network_fd;
    epoll_ctl(epfd, EPOLL_CTL_ADD, network_fd, &ev);

    ev.events = EPOLLIN;
    ev.data.fd = lvgl_tfd;
    epoll_ctl(epfd, EPOLL_CTL_ADD, lvgl_tfd, &ev);

    struct epoll_event events[MAX_EVENTS];
    int running = 1;

    while (running) {
        /* 计算到下一次 LVGL 事件的时间，作为 epoll 超时 */
        uint32_t time_till_next = lv_timer_handler();
        int timeout = (time_till_next < LVGL_PERIOD_MS)
                    ? time_till_next : LVGL_PERIOD_MS;

        int nfds = epoll_wait(epfd, events, MAX_EVENTS, timeout);

        for (int i = 0; i < nfds; i++) {
            int fd = events[i].data.fd;

            if (fd == lvgl_tfd) {
                /* LVGL 定时器触发 */
                uint64_t exp;
                read(lvgl_tfd, &exp, sizeof(exp));
                lv_tick_inc(LVGL_PERIOD_MS);
            }
            else if (fd == touch_fd) {
                /* 触摸屏输入 —— LVGL 输入驱动会读取 */
                /* 实际实现在 input driver 的 read 回调中 */
            }
            else if (fd == network_fd) {
                /* 网络数据到达 */
                char buf[1024];
                int n = read(network_fd, buf, sizeof(buf));
                if (n > 0) {
                    /* 处理网络数据，更新 UI */
                    handle_network_data(buf, n);
                }
            }
        }
    }

    close(lvgl_tfd);
    close(epfd);
    return 0;
}
```

**架构示意**：

```
┌─────────────────────────────────────────────┐
│              epoll_wait()                    │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐    │
│  │touch_fd  │ │net_fd    │ │lvgl_tfd  │    │
│  │(EPOLLIN) │ │(EPOLLIN) │ │(EPOLLIN) │    │
│  └────┬─────┘ └────┬─────┘ └────┬─────┘    │
│       │            │            │           │
│       ▼            ▼            ▼           │
│  read touch   read network  lv_tick_inc()   │
│  → lv_indev   → update UI   lv_timer_handler│
└─────────────────────────────────────────────┘
```

## 2.6 多线程安全 {#26-多线程安全}

**LVGL 不是线程安全的**。所有对 LVGL API 的调用必须在同一个线程中完成（通常是主线程/UI 线程）。在 i.MX6ULL 这样的多核 ARM 平台上，后台线程需要更新 UI 时，必须通过安全的机制进行。

### 2.6.1 为什么 LVGL 不是线程安全的

LVGL 内部大量使用全局状态和链表操作，没有内置互斥锁。如果多个线程同时调用 LVGL API，会导致：

- 链表损坏（双重释放、野指针）
- 显示撕裂（半帧更新）
- 样式/对象状态不一致

### 2.6.2 互斥锁保护方案

最基本的方案是用互斥锁包裹 LVGL 调用：

```c
#include <pthread.h>
#include "lvgl.h"

static pthread_mutex_t lvgl_mutex = PTHREAD_MUTEX_INITIALIZER;

/* 在后台线程中更新 UI 时使用 */
void lvgl_lock(void)
{
    pthread_mutex_lock(&lvgl_mutex);
}

void lvgl_unlock(void)
{
    pthread_mutex_unlock(&lvgl_mutex);
}

/* 在主循环中使用 */
while (running) {
    lvgl_lock();
    lv_timer_handler();
    lvgl_unlock();
    usleep(5000);
}

/* 在后台线程中使用（不推荐直接调用 LVGL API） */
void * bg_thread(void * arg)
{
    while (1) {
        int value = read_sensor();
        lvgl_lock();
        lv_label_set_text_fmt(sensor_label, "Temp: %d°C", value);
        lvgl_unlock();
        sleep(1);
    }
    return NULL;
}
```

**问题**：这种方式虽然能保护数据一致性，但存在以下缺点：

- 后台线程持有锁时，主线程的 `lv_timer_handler()` 被阻塞，导致 UI 卡顿
- 死锁风险增加
- 违反了 UI 框架的事件驱动设计

### 2.6.3 推荐方案：消息机制

LVGL v8.x 提供了 `lv_msg` 消息系统，这是从后台线程安全更新 UI 的推荐方式：

```c
#include "lvgl.h"
#include <pthread.h>
#include <unistd.h>

/* 定义消息 ID */
enum {
    MSG_SENSOR_TEMP = 1,
    MSG_NETWORK_STATUS,
    MSG_BATTERY_LEVEL,
};

/* ====== UI 线程（主线程）====== */

static void on_sensor_temp(lv_event_t * e)
{
    lv_obj_t * label = lv_event_get_user_data(e);
    int * temp = (int *)lv_event_get_param(e);
    if (temp) {
        lv_label_set_text_fmt(label, "Temperature: %d°C", *temp);
    }
}

void setup_ui(void)
{
    lv_obj_t * scr = lv_scr_act();
    lv_obj_t * label = lv_label_create(scr);
    lv_label_set_text(label, "Temperature: --");
    lv_obj_center(label);

    /* 订阅消息 */
    lv_obj_add_event_cb(label, on_sensor_temp, LV_EVENT_MSG_RECEIVED, NULL);
    lv_msg_subscribe_obj(MSG_SENSOR_TEMP, label, NULL);
}

/* ====== 后台线程（传感器读取）====== */

static void * sensor_thread(void * arg)
{
    (void)arg;
    while (1) {
        int temperature = read_temperature_sensor();  /* 读取传感器 */

        /* 通过 lv_msg 发送消息，主线程会在下次 lv_timer_handler 时处理 */
        lv_msg_send(MSG_SENSOR_TEMP, &temperature);

        sleep(1);
    }
    return NULL;
}

/* ====== 主函数 ====== */

int main(void)
{
    /* ... LVGL 初始化 ... */
    lv_init();
    /* ... 显示驱动、输入驱动初始化 ... */

    setup_ui();

    /* 启动后台线程 */
    pthread_t tid;
    pthread_create(&tid, NULL, sensor_thread, NULL);

    /* 主循环 */
    while (1) {
        lv_timer_handler();
        usleep(5000);  /* 5ms */
        lv_tick_inc(5);
    }

    return 0;
}
```

**工作原理**：

1. 后台线程调用 `lv_msg_send()` 发送消息
2. `lv_msg_send()` 只是将消息入队（这个操作是原子的/快速的）
3. 主线程在 `lv_timer_handler()` 中处理消息队列
4. 订阅者回调在 UI 线程的上下文中执行，可以安全调用任何 LVGL API

### 2.6.4 另一种方案：自定义事件 + 信号量

如果不想使用 `lv_msg`，也可以通过 Linux 原语（eventfd）实现线程间通知：

```c
#include "lvgl.h"
#include <pthread.h>
#include <sys/eventfd.h>
#include <unistd.h>

/* 线程安全的数据交换结构 */
typedef struct {
    int temperature;
    int humidity;
    pthread_mutex_t lock;
} shared_data_t;

static shared_data_t shared = {
    .temperature = 0,
    .humidity = 0,
    .lock = PTHREAD_MUTEX_INITIALIZER,
};

static int notify_fd = -1;
static lv_obj_t * temp_label = NULL;
static lv_obj_t * humid_label = NULL;

/* LVGL 定时器回调 —— 检查是否有新数据 */
static void check_update_cb(lv_timer_t * timer)
{
    uint64_t count;
    /* 非阻塞读取 eventfd */
    if (read(notify_fd, &count, sizeof(count)) == sizeof(count)) {
        /* 有更新通知，读取共享数据 */
        pthread_mutex_lock(&shared.lock);
        int temp = shared.temperature;
        int humid = shared.humidity;
        pthread_mutex_unlock(&shared.lock);

        /* 在 UI 线程中安全更新 */
        lv_label_set_text_fmt(temp_label, "Temp: %d°C", temp);
        lv_label_set_text_fmt(humid_label, "Humid: %d%%", humid);
    }
}

/* 后台线程 */
static void * sensor_thread(void * arg)
{
    (void)arg;
    while (1) {
        int t = read_temperature_sensor();
        int h = read_humidity_sensor();

        /* 写入共享数据 */
        pthread_mutex_lock(&shared.lock);
        shared.temperature = t;
        shared.humidity = h;
        pthread_mutex_unlock(&shared.lock);

        /* 通知 UI 线程 */
        uint64_t one = 1;
        write(notify_fd, &one, sizeof(one));

        sleep(1);
    }
    return NULL;
}

/* 初始化 */
void init_sensor_monitor(void)
{
    notify_fd = eventfd(0, EFD_NONBLOCK);

    lv_obj_t * scr = lv_scr_act();
    temp_label = lv_label_create(scr);
    humid_label = lv_label_create(scr);
    lv_obj_align(temp_label, LV_ALIGN_CENTER, 0, -20);
    lv_obj_align(humid_label, LV_ALIGN_CENTER, 0, 20);

    /* 创建 100ms 轮询定时器 */
    lv_timer_create(check_update_cb, 100, NULL);

    /* 启动后台线程 */
    pthread_t tid;
    pthread_create(&tid, NULL, sensor_thread, NULL);
}
```

### 2.6.5 多线程安全最佳实践总结

| 方案 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| **互斥锁** | 简单场景，偶尔更新 | 实现简单 | 可能阻塞 UI，死锁风险 |
| **lv_msg** | 推荐使用 | LVGL 原生，线程安全 | 需要 v8.3+ |
| **eventfd + 定时器** | 需要与 epoll 集成 | 与 Linux 生态契合 | 实现稍复杂 |
| **消息队列** | 复杂数据交换 | 灵活 | 需要自行实现 |

**黄金法则**：尽可能让所有 LVGL API 调用都发生在 UI 线程中。

| 概念 | 关键 API | 要点 |
